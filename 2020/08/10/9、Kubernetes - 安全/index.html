9.K8s-安全学习

Posted on 2020-08-10

api service 是整个k8s中很重要的组件，他的安全性稳定性影响着整个集群的鲁棒性。

机制说明

![image-20200810101748781](9、Kubernetes - 安全.assets/image-20200810101748781.png)

认证（Authentication）

简单的理解就是当pod或者用户需要访问 API server 时需要进行身份的识别认证。有以下几种认证方式：

  • HTTP Token：通过一个Token(或者字符串)来识别用户。假如“1234”才是合法用户，你拿着一个1234来认证，那我就认为你是合法用户

    • HTTP token 的认证是用一个*很长的* 特殊编码** 方式的并且 *难以被模仿*** 的字符串-Token来表达客户的一种方式。每一个Token对应一个用户存储在 API server 能访问的文件中。当客户端发起 API 调用请求时，需要在HTTP Header 里放入 Token ___ETCD????
    • 这种放在文件中，不安全

  • HTTP Base: 通过用户名+密码的方式进行认证

    • 将用户名+密码用base64方式进行编码，然后将编码后的字符串放在 HTTP Request 中的 Heather Authorization 域里发送给服务端，服务端接收后进行解码，获取用户名+密码
    • 不安全费劲

    前两种方法安全性并不是那么高，都给了服务器认证客户端的机会，但是客户端不能认证服务器，服务器可以认证客户端是否合理合法，但是客户端没办法认证服务器是否合理合法。

  • HTTPS 证书：最严格的认证机制

    • 基于 CA 根证书签名的客户端身份认证方式，k8s中双向认证。。。
    • 最常用
